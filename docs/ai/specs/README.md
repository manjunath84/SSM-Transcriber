# `docs/ai/specs/`

Focused feature specs live here when a change is large enough to deserve its
own design doc.

Guidelines:

- Keep one spec per chunk of work, not one giant product document.
- Include problem, scope, constraints, acceptance criteria, and out-of-scope.
- Link back to `docs/PLAN.md` when phase ordering or F1–F8 contracts matter.
- Do not turn this folder into a second roadmap; the phased plan still lives in
  `docs/PLAN.md`.
